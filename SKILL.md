## Core Architecture Principle (Non-Negotiable)

The website is a read-only display layer for reports generated by the PMTE data pipeline. It does not perform analysis. All computation happens in PMTE. The website only displays, filters, and gates access to pre-generated output. This keeps the site fast, cheap to host, and architecturally clean forever.

## Stripe Configuration

### Products (create before Phase 3)
- Analyst Access: monthly ($12) + annual ($99)
- Professional Access: monthly ($29) + annual ($249)
- Institutional Access: monthly ($199) — deferred

### Metadata Convention (CRITICAL)
Each Stripe product must have a metadata field: `tier = [code value]`
```
Analyst Access product:       metadata.tier = 'analyst_access'
Professional Access product:  metadata.tier = 'professional_access'
Institutional Access product: metadata.tier = 'institutional_access'
```
Webhook handler reads `event.data.object.metadata.tier` and writes that exact string to `users.tier` in Supabase. No hardcoded mapping in code. On `subscription.deleted`: set `users.tier = 'public_access'`.

### Webhook Events — Handle ALL Three
```
checkout.session.completed     → upgrade tier in Supabase
customer.subscription.deleted  → downgrade to public_access
invoice.payment_failed         → flag account, notify user
```

## Federal Money Board Ticker (Locked Design)

The homepage hero is a **Federal Money Board** — a live counter showing federal dollars obligated this fiscal year. Primary metric counts UP (money moved), not down.

### Visual Design (Locked — Do Not Deviate)
```
Background:    paper #f5f0e8 with noise texture
Board border:  2px solid #1a1a1a
Header bar:    #1a1a1a bg, #f5f0e8 text, red live dot #c0392b
Budget strip:  #2c2c2c bg, #d4c5a9 / #f5f0e8 text
Main number:   IBM Plex Mono, clamp(30px,7vw,56px), #c0392b red
Sub label:     Playfair Display italic, #2c2c2c
Bar:           fills left-to-right, #c0392b, track #d4c5a9
Secondary:     '$X.XXT unobligated budget remains' in #8b7355
Stats cells:   #f5f0e8 bg, #1a1a1a values, #8b7355 labels
State cells:   #ede8df bg
Footer bar:    #1a1a1a bg, #8b7355 text
Red elements:  ONLY main number, bar fill, live dot, state arrow, CTA link
```

### Board Sections (Top to Bottom)
1. Header bar — 'Federal Money Board' left, 'FY 2026 · Oct 1, 2025–Sep 30, 2026 · N days remaining' right
2. Budget strip — 'FY 2026 Authorized: $6.752 Trillion' + Discretionary/Mandatory/Interest breakdown
3. Main block — eyebrow 'FEDERAL DOLLARS MOVED · FY 2026', big red number counting up, italic label, committed bar, '$X.XXT unobligated remains'
4. Stats row — Contracts today / Grants today / Largest award / States receiving (2-col mobile, 4-col desktop)
5. State row — state share left, 'Select your state ↓' right (arrow in #c0392b)
6. Footer bar — source attribution + last refresh timestamp

### ticker_state.json Schema
```json
{
  "fy": "2026",
  "fy_start": "2025-10-01",
  "fy_end": "2026-09-30",
  "authorized_budget": 6752000000000,
  "baseline_obligated": 3417211992340,
  "baseline_timestamp": "2026-03-14T02:00:00Z",
  "rate_per_second": 212600,
  "last_refresh": "Mar 14, 2026 · 2:00 AM ET",
  "state_totals": { "MI": 42118900221 },
  "today": {
    "contracts": 14200000000,
    "grants": 9600000000,
    "largest_award": 2100000000,
    "states_receiving": 38
  }
}
```

### Ticker Logic
```
obligated = baseline_obligated + (rate_per_second * seconds_since_baseline)
remainder = authorized_budget - obligated
committedPct = (obligated / authorized_budget) * 100
Updates every 1 second via setInterval
```

**Phase 1:** hardcoded mock values labeled as mock  
**Phase 4:** wire to `website/data/ticker_state.json`

### Ticker Freshness
- `ticker_state.json` redeployed to Vercel after every Friday PMTE refresh
- Footer shows `last_refresh` timestamp from file — never fake or hide it
- Cache TTL for `ticker_state.json` fetch: max 1 hour
